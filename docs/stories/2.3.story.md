# Story 2.3 — Dashboard de Métricas Admin
**Epic:** EPIC-2 CMS Admin  
**Status:** Ready  
**Complexity:** M (3 pts)  
**Priority:** P1

## Descrição
Como membro da equipe Carmem, quero visualizar métricas de engajamento da plataforma — aulas mais assistidas, taxas de conclusão e usuários ativos — para tomar decisões de conteúdo.

## Acceptance Criteria

**Given** admin acessa `/admin/dashboard`  
**When** visualizar métricas  
**Then:**
- [ ] Card: Total de médicos cadastrados
- [ ] Card: Médicos ativos nos últimos 7 dias (via last_seen_at)
- [ ] Card: Médicos ativos nos últimos 30 dias
- [ ] Card: % de médicos com NPS respondido
- [ ] Tabela: Top 5 aulas mais assistidas (por conclusão count)
- [ ] Tabela: Taxa de conclusão por trilha (% de médicos que completaram)
- [ ] Card: Score NPS geral (média das respostas)

## Scope
**IN:**
- Dashboard com queries agregadas no Supabase
- KPIs definidos no PRD seção 10

**OUT:**
- Gráficos animados / charts (usar tabelas simples por ora)
- Export para CSV

## Dependencies
- Story 1.2 (dados no banco), Story 2.2 (usuários criados)

## Definition of Done
- [ ] Dashboard carrega dados reais (não mocked)
- [ ] `npm run lint` passa
