# Story 2.2 — Gestão de Usuários + Convite de Médicos
**Epic:** EPIC-2 CMS Admin  
**Status:** Ready  
**Complexity:** M (5 pts)  
**Priority:** P0

## Descrição
Como membro da equipe Carmem, quero adicionar médicos à plataforma por e-mail, com disparo automático do e-mail de boas-vindas, e poder ver o progresso individual de cada médico.

## Acceptance Criteria

**Given** admin acessa `/admin/usuarios`  
**When** gerenciar médicos  
**Then:**
- [ ] Listar todos os médicos com: nome, e-mail, data de cadastro, progresso geral (%), last_seen_at, nps_answered
- [ ] Formulário "Adicionar médico": nome + e-mail → cria usuário no Supabase Auth + perfil em profiles + dispara e-mail de boas-vindas via Resend
- [ ] Ver detalhe do médico: progresso por trilha, aulas concluídas, badges, histórico
- [ ] Revogar acesso: desativar conta (não deletar dados)
- [ ] Busca de médico por nome ou e-mail na listagem

## Scope
**IN:**
- Listagem de médicos com filtros básicos
- Adição de médico com e-mail de boas-vindas
- Detalhe do progresso individual
- Revogação de acesso

**OUT:**
- Edição de dados do médico pelo admin (médico edita o próprio perfil)
- Export CSV (V2)

## Dependencies
- Story 1.2 (schema), Story 1.3 (Supabase Auth), Story 4.2 (Resend e-mail)

## Definition of Done
- [ ] Admin adiciona médico e e-mail de boas-vindas é disparado
- [ ] Progresso do médico visível no detalhe
- [ ] `npm run typecheck` passa

## Dev Notes

- Service role key: apenas no servidor, nunca exposta ao cliente
