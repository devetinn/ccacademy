# Story 3.3 — Player de Vídeo + Progresso Salvo
**Epic:** EPIC-3 Core Learning  
**Status:** Ready  
**Complexity:** L (8 pts)  
**Priority:** P0

## Descrição
Como médico cliente, quero assistir vídeos com controle de velocidade, ter meu progresso salvo automaticamente e retomar de onde parei na próxima visita.

## Acceptance Criteria

**Given** médico acessa `/aula/[id]`  
**When** interagir com o player  
**Then:**
- [ ] Player carrega o vídeo (YouTube embed via react-player)
- [ ] Controle de velocidade: 0.75x / 1x / 1.25x / 1.5x funcionando
- [ ] Progresso salvo a cada 5 segundos (debounced) em `lesson_progress.watched_pct`
- [ ] Ao reabrir a aula: player retoma do ponto onde parou (usa `lesson_progress.watched_pct`)
- [ ] 80% assistido → `lesson_progress.completed = true` automaticamente
- [ ] Aula marcada com ícone de concluída na listagem da trilha
- [ ] Header da aula: título + trilha de origem (breadcrumb)
- [ ] Botão "Próxima aula" ao final (navega para a próxima em ordem)

## Scope
**IN:**
- react-player com YouTube embed
- Salvamento de progresso (watched_pct + completed)
- Retomada de onde parou
- Controle de velocidade
- Lógica de conclusão automática (80%)

**OUT:**
- Materiais de apoio (Story 3.4)
- Q&A (Story 4.1)

## Dependencies
- Story 1.2 (lesson_progress table), Story 3.2 (navegação de trilha)

## Definition of Done
- [ ] Progresso persistido no banco (confirmado via Supabase Table Editor)
- [ ] Retomada funcionando após fechar e reabrir
- [ ] 80% → concluída automática funcionando
- [ ] Velocidades 0.75x a 1.5x funcionando
- [ ] `npm run typecheck` passa

## Dev Notes
- react-player: usar `onProgress` callback para capturar `played` (0-1) → converter para %
- Debounce de 5s no save para não sobrecarregar o banco
- YouTube embed: `?enablejsapi=1&rel=0&modestbranding=1` nos params
- `watched_pct` é INTEGER 0-100 — usar `Math.floor(played * 100)`
