# Story 3.5 — Certificado Digital Automático
**Epic:** EPIC-3 Core Learning  
**Status:** Ready  
**Complexity:** M (3 pts)  
**Priority:** P1

## Descrição
Como médico cliente, quero receber automaticamente um certificado digital quando concluir 100% de uma trilha, como reconhecimento do meu aprendizado.

## Acceptance Criteria

**Given** médico conclui a última aula de uma trilha  
**When** o progresso for 100%  
**Then:**
- [ ] Certificado inserido em `certificates` (user_id + trail_id + issued_at)
- [ ] Página `/certificado/[id]` exibe o certificado com: nome do médico, nome da trilha, data de conclusão, logo Carmem
- [ ] E-mail enviado com link para o certificado
- [ ] Botão "Baixar certificado" gera PDF (via html-to-pdf ou CSS print)

## Scope
**IN:**
- Lógica de detecção de 100% (trigger ou verificação pós-update de lesson_progress)
- Página do certificado
- Geração de PDF simples

**OUT:**
- Certificado com assinatura digital/blockchain
- Integração com CFM

## Dependencies
- Story 1.2 (certificates table), Story 3.3 (lesson_progress), Story 4.2 (e-mail)

## Definition of Done
- [ ] Certificado criado automaticamente ao 100%
- [ ] Página renderiza com dados reais
- [ ] `npm run typecheck` passa
