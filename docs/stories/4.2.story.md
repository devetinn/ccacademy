# Story 4.2 — E-mails Transacionais via Resend
**Epic:** EPIC-4 Engagement  
**Status:** Ready  
**Complexity:** M (5 pts)  
**Priority:** P0

## Descrição
Como sistema, preciso enviar e-mails transacionais com identidade visual da Carmem Cavalcante nos três eventos principais: boas-vindas, resposta de Q&A e desbloqueio de workshop.

## Acceptance Criteria

**Given** cada evento ocorre  
**When** o trigger for disparado  
**Then:**

**E-mail 1 — Boas-vindas:**
- [ ] Disparado quando admin adiciona novo médico (Story 2.2)
- [ ] Conteúdo: nome do médico, link de acesso magic link, trilha recomendada
- [ ] Template com logo Carmem, azul institucional, assinatura da equipe

**E-mail 2 — Dúvida respondida:**
- [ ] Disparado quando admin publica resposta no Q&A (Story 4.1)
- [ ] Conteúdo: texto da pergunta + resposta + link direto para a aula

**E-mail 3 — Workshop desbloqueado:**
- [ ] Disparado após submissão do NPS (Story 5.1)
- [ ] Conteúdo: parabéns + link para assistir o workshop completo

**Todos os e-mails:**
- [ ] Enviados via Resend API
- [ ] Template HTML responsivo com identidade visual Carmem
- [ ] From: `academy@carmemcavalcante.com.br` (ou similar)
- [ ] Entrega verificada (não caindo em spam)

## Scope
**IN:**
- 3 templates HTML para Resend
- Funções de envio em `lib/resend.ts`
- Integração com triggers dos outros épicos

**OUT:**
- E-mails de marketing/newsletter
- E-mail de certificado (pode ser estendido aqui ou na 3.5)

## Dependencies
- Story 1.1 (RESEND_API_KEY no env), Story 2.2, Story 4.1, Story 5.1

## Definition of Done
- [ ] 3 templates funcionando em preview Resend
- [ ] E-mails chegando na caixa de entrada (não spam)
- [ ] `npm run typecheck` passa
