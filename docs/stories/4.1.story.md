# Story 4.1 — Q&A por Aula (Perguntar + Responder)
**Epic:** EPIC-4 Engagement  
**Status:** Ready  
**Complexity:** M (5 pts)  
**Priority:** P0

## Descrição
Como médico cliente, quero fazer perguntas sobre o conteúdo de cada aula e receber uma resposta da equipe Carmem. Como admin, quero ver todas as perguntas e respondê-las de forma eficiente.

## Acceptance Criteria

**Given** médico acessa `/aula/[id]`  
**When** usar a seção de Q&A  
**Then:**
- [ ] Campo de texto livre "Sua dúvida" com botão "Enviar"
- [ ] Pergunta salva em `lesson_questions` com `is_public=true` por default
- [ ] Perguntas respondidas exibidas como FAQ (pergunta + resposta + data)
- [ ] Perguntas próprias não respondidas aparecem como "Aguardando resposta"
- [ ] Perguntas públicas de outros médicos visíveis para todos

**Given** admin acessa `/admin/perguntas`  
**When** gerenciar Q&A  
**Then:**
- [ ] Listar todas as perguntas abertas (sem resposta) com: aula, médico, data, texto
- [ ] Campo de resposta com editor de texto simples (textarea)
- [ ] Botão "Publicar resposta" → salva em `lesson_questions.answer` + dispara e-mail
- [ ] Toggle "Pública/Privada" por pergunta

## Scope
**IN:**
- QASection component na página da aula
- Painel admin `/admin/perguntas`
- Notificação por e-mail ao publicar resposta (integração Story 4.2)

**OUT:**
- Threads aninhadas / likes nas perguntas
- Sistema de moderação avançado

## Dependencies
- Story 1.2, Story 3.3 (página da aula), Story 4.2 (e-mail)

## Definition of Done
- [ ] Fluxo completo: médico pergunta → admin responde → médico recebe e-mail
- [ ] Perguntas públicas visíveis para todos
- [ ] `npm run typecheck` passa
