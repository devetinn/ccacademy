# Story 1.3 — Autenticação Magic Link + Middleware de Rotas
**Epic:** EPIC-1 Foundation  
**Status:** Ready  
**Complexity:** M (5 pts)  
**Priority:** P0

## Descrição
Como médico cliente, quero receber um link por e-mail e acessar a Academy com um clique, sem criar senha. Como sistema, preciso proteger todas as rotas de `/academy/*` e `/admin/*` contra acesso não autenticado.

## Acceptance Criteria

**Given** um médico tem e-mail cadastrado no Supabase  
**When** solicitar acesso  
**Then:**
- [ ] Página `/login` com campo de e-mail e botão "Enviar link de acesso"
- [ ] Supabase Auth envia magic link ao e-mail informado
- [ ] Médico clica no link e é autenticado automaticamente
- [ ] Perfil criado automaticamente em `profiles` no primeiro acesso (via Supabase trigger ou server action)
- [ ] Redirecionamento: após login → `/` (home da Academy)
- [ ] Sessão persistida (refresh token gerenciado pelo `@supabase/ssr`)

**Given** uma rota protegida é acessada sem autenticação  
**When** o middleware interceptar  
**Then:**
- [ ] Redirecionar para `/login`

**Given** um usuário com `role = 'student'` acessa `/admin/*`  
**When** o middleware interceptar  
**Then:**
- [ ] Retornar 403 ou redirecionar para `/`

**Given** um usuário com `role = 'admin'` acessa `/admin/*`  
**When** autenticado  
**Then:**
- [ ] Acesso liberado normalmente

## Scope
**IN:**
- Página `/login` (magic link)
- `middleware.ts` (proteção `/academy/*` + `/admin/*`)
- `lib/supabase/middleware.ts` (updateSession)
- Criação automática de profile no primeiro login

**OUT:**
- UI da home da Academy (Story 3.1)
- UI do admin (Epic 2)
- Registro manual de médicos pelo admin (Story 2.2)

## Dependencies
- Story 1.1 (Supabase client configurado)
- Story 1.2 (tabela profiles existente)

## Definition of Done
- [ ] Magic link funciona end-to-end (envio + clique + autenticação)
- [ ] Rota `/academy/` sem login redireciona para `/login`
- [ ] Rota `/admin/` com role=student redireciona para `/`
- [ ] Perfil criado automaticamente na `profiles` no primeiro acesso
- [ ] `npm run typecheck` passa

## Dev Notes
- Usar `createServerClient` do `@supabase/ssr` no middleware
- O `updateSession` deve estar no middleware para refresh automático de tokens
- Para criar o profile automaticamente: usar Supabase Database Trigger em `auth.users` INSERT
  → função `handle_new_user()` que insere em `profiles`
- Role de admin: definir via Supabase Dashboard (SQL: UPDATE profiles SET role='admin' WHERE email='...')
