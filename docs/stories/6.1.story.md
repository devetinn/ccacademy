# Story 6.1 — QA em Dispositivos Móveis + Correções
**Epic:** EPIC-6 Launch  
**Status:** Ready  
**Complexity:** M (5 pts)  
**Priority:** P0

## Descrição
Como equipe de desenvolvimento, precisamos validar toda a plataforma em dispositivos móveis reais (iPhone Safari e Android Chrome) e corrigir quaisquer problemas de UX antes do go-live.

## Acceptance Criteria

**Given** a plataforma está em staging (Vercel preview)  
**When** executar o QA em dispositivos reais  
**Then:**
- [ ] Fluxo de login (magic link) funciona em iPhone Safari
- [ ] Fluxo de login funciona em Android Chrome
- [ ] Home com trilhas renderiza corretamente em 375px
- [ ] Player de vídeo funciona em mobile (YouTube embed)
- [ ] Materiais de apoio: download e PDF preview funcionam no mobile
- [ ] Q&A: envio de pergunta funciona em teclado virtual
- [ ] NPS Gate: formulário funciona em mobile
- [ ] Certificado renderiza corretamente
- [ ] Nenhum elemento ultrapassa a largura da tela (overflow-x: hidden)
- [ ] Touch targets mínimos de 44px em todos os botões
- [ ] Tempo de carregamento da home < 3 segundos em 4G

## Scope
**IN:**
- QA de todos os fluxos da Academy (não admin)
- Correções de layout mobile
- Performance básica

**OUT:**
- Testes automatizados (Playwright — V2)
- Load testing

## Dependencies
- EPIC-1 a EPIC-5 completos

## Definition of Done
- [ ] Checklist de QA mobile 100% concluído
- [ ] Todos os bugs críticos corrigidos
- [ ] Relatório de QA salvo em `docs/qa/launch-qa-report.md`
