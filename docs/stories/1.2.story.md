# Story 1.2 — Schema do Banco de Dados + Migrations + RLS
**Epic:** EPIC-1 Foundation  
**Status:** Ready  
**Complexity:** M (5 pts)  
**Priority:** P0

## Descrição
Como data engineer, preciso aplicar o schema completo do banco de dados no Supabase com todas as tabelas, relacionamentos, triggers e políticas RLS conforme especificado no PRD seção 6.

## Acceptance Criteria

**Given** o projeto Supabase está ativo  
**When** as migrations forem aplicadas  
**Then:**
- [ ] Tabela `profiles` criada com campos: id, full_name, email, crm, avatar_url, points, nps_answered, role, created_at, last_seen_at
- [ ] Tabela `trails` criada (id, title, description, cover_image, estimated_min, order_index, is_active, created_at)
- [ ] Tabela `modules` criada (id, trail_id FK, title, order_index)
- [ ] Tabela `lessons` criada (id, module_id FK, title, description, video_url, duration_sec, is_locked, lock_condition, order_index, created_at)
- [ ] Tabela `lesson_attachments` criada (id, lesson_id FK, file_name, file_url, file_type CHECK, file_size_kb, download_count, created_at)
- [ ] Tabela `attachment_downloads` criada (user_id FK, attachment_id FK, downloaded_at)
- [ ] Tabela `lesson_progress` criada com UNIQUE(user_id, lesson_id)
- [ ] Tabela `lesson_questions` criada (pergunta + resposta + is_public)
- [ ] Tabela `nps_responses` criada (score CHECK 0-10, comment, created_at)
- [ ] Tabela `badges` e `user_badges` criadas com UNIQUE(user_id, badge_id)
- [ ] Tabela `certificates` criada com UNIQUE(user_id, trail_id)
- [ ] Trigger `after_nps_insert` → `set_nps_answered()` funcionando
- [ ] RLS ativo nas tabelas: profiles, lesson_progress, lesson_questions, nps_responses
- [ ] Políticas RLS criadas: own_profile, own_progress, own_nps, qa_read, qa_insert
- [ ] Migration salva em `supabase/migrations/001_initial_schema.sql`

## Scope
**IN:**
- Todas as tabelas do PRD seção 6
- Todos os relacionamentos FK
- Trigger `after_nps_insert`
- Políticas RLS conforme PRD
- Arquivo de migration versionado

**OUT:**
- Dados seed (Story 6.2)
- Supabase Storage buckets (Story 3.4)
- Edge Functions

## Dependencies
- Story 1.1 concluída (projeto Next.js com Supabase configurado)

- [ ] RLS ativo e políticas aplicadas
- [ ] Migration commitada em `supabase/migrations/`
